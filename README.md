# CS360 Module Eight Journal reflection
Developing my mobile application for Project Three helped me understand how all the different components of an Android app come together to meet real user needs. The goal of my app was to give users a simple, reliable way to track their daily weight, set a personal goal, and receive optional SMS notifications when they reached that goal. This meant designing a user‑centered interface that was easy to navigate, visually clear, and focused on helping users complete their tasks without confusion. The screens I created including login, dashboard, daily weight entry, goal‑setting, SMS permission, and history were all structured to guide users step‑by‑step through the app’s core functions. I kept users in mind by using clean layouts, clear labels, and straightforward workflows, which made the design successful and intuitive.

When I began coding the app, I approached the process by breaking the project into smaller, manageable pieces. I focused first on building the UI layouts, then connected each screen to its Activity, and finally integrated the database and SMS functionality. I used strategies such as incremental testing, version control, and consistent naming conventions to stay organized. These techniques helped me troubleshoot issues more easily and will be valuable in future development projects. Testing was an essential part of the process, especially because Android development involves many moving parts Activities, permissions, emulators, and different SDK versions. Running the app frequently allowed me to catch errors early, understand how the UI behaved on different devices, and ensure that the database and SMS features worked correctly. This process revealed how important it is to test both functionality and user experience throughout development.

Looking back at the full design and development cycle, I had to innovate several times to overcome challenges, especially when dealing with emulator issues, SDK mismatches, and runtime permissions. These obstacles pushed me to research solutions, experiment with different approaches, and stay persistent. One area where I was particularly successful was integrating the SQLite database with the UI. I demonstrated strong understanding of data flow by connecting user input to stored records and displaying that data in a RecyclerView history screen. This component showcased my ability to combine backend logic with user‑centered design. Overall, Project Three strengthened my confidence in building functional mobile apps and gave me practical experience that I can apply to future coursework, personal projects, and eventually professional development.
